# Spec 31 ŌĆö Onboarding, Organizations, & Permissions DX

## Overview

Improves the developer experience and end-user onboarding for VEX CMS projects. Five features: (1) fix media collection field autocomplete in access permissions, (2) landing page sign-in/sign-up flow with auto-admin for first user, (3) guided admin panel onboarding tour via driver.js, (4) `admin.onboarding` config option + user doc flag, (5) multi-tenant organizations in create CLI + template.

## Design Decisions

- **First-user auto-admin via Convex mutation:** A post-signup mutation atomically checks if any admin exists. If not, promotes the new user. A `vex_admin_bootstrapped` flag (checked via user count with admin role) prevents re-triggering ŌĆö even if the admin is later deleted, no new auto-admins are created.
- **Onboarding tour via driver.js:** ~15KB gzipped, loaded only in admin panel. Uses `data-tour` attributes on key UI elements for stable targeting. All-or-nothing dismissal ŌĆö once dismissed, marked complete on user document. Users can restart from user menu dropdown.
- **Onboarding flag on user document:** `vex_onboarding_complete: boolean` field on user doc. Persists across devices. Checked on admin panel mount.
- **Organizations via CLI prompt + `--orgs` flag:** Interactive prompt during scaffold. Also accepts `--orgs` flag to skip prompt. When enabled, adds Better Auth organizations plugin. Tables auto-generated by `vexBetterAuth()` ŌĆö no manual schema changes.
- **Media field autocomplete fix:** `resolveMediaCollection()` converts `VexMediaCollection` ŌåÆ `VexCollection`, losing the `DefaultMediaFieldKeys` type info. Fix by adding a `_isMedia` marker to the resolved collection and updating `ExtractFieldKeys` to check for it.
- **Field permissions `{ mode, fields }` already works:** Both static and callback forms already support `{ mode: "allow" | "deny", fields: FieldKey[] }`. No changes needed to the permission format itself.

## Out of Scope

- Admin settings page (onboarding restart goes in user menu dropdown)
- Organization-scoped document filtering in queries
- Organization switcher UI in admin panel
- Invitation flow UI for organizations
- Custom onboarding tour steps config API

```typescript
export type ExtractFieldKeys<T> = T extends VexCollection<infer TFields, infer TExtraKeys>
  ? T extends { _isMedia: true }
    ? (keyof TFields & string) | (TExtraKeys & string) | DefaultMediaFieldKeys
    : (keyof TFields & string) | (TExtraKeys & string)
  : T extends VexMediaCollection<infer TFields>
    ? (keyof TFields & string) | DefaultMediaFieldKeys
    : T extends VexGlobal<infer TFields>
      ? keyof TFields & string
      : never;
```
